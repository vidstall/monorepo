# Roadmap: DVConf

## Overview

This milestone takes DVConf from "Phase 1 code exists but unvalidated" to "all on-chain registries deployed and off-chain daemons running against testnet." Phase 1 validates and deploys the existing foundation contracts. Phase 2 builds the five registry modules that the room lifecycle and economic layers will depend on. Phase 3 builds the off-chain daemons (signaling, control plane, validator) that consume registry events and interact with the chain.

## Phases

**Goal**: Five independent registry modules exist as shared objects on testnet, each with full test coverage, event emission, and paused-flag enforcement

**Requirements**: REG-01, REG-02, REG-03, REG-04, REG-05, REG-06, REG-07, REG-08, REG-09, REG-10, REG-11, REG-12, REG-13, REG-14, REG-15
**Success Criteria** (what must be TRUE):
  1. RoomManager can create rooms with configurable rules (min_relay, min_cp, min_validator) and tracks active room count
  2. ControlPlaneRegistry supports CP registration, heartbeat liveness, and room assignment tracking

### Phase 3: Off-chain Daemons
**Goal**: Signaling, Control Plane, and Validator daemons run as Node.js processes in a pnpm monorepo, subscribe to on-chain events via queryEvents polling, and interact with deployed registry contracts
**Depends on**: Phase 2
**Requirements**: DAEMON-01, DAEMON-02, DAEMON-03, DAEMON-04, DAEMON-05, DAEMON-06, DAEMON-07, DAEMON-08, DAEMON-09, DAEMON-10, DAEMON-11, DAEMON-12
**Success Criteria** (what must be TRUE):
  1. Signaling node exchanges WebRTC ICE candidates over WebSocket without depending on chain state
  2. CP daemon subscribes to Sui events, runs relay scoring using on-chain data (reputation, RTT, load, stake, region), and sends heartbeats on-chain with exponential backoff on failures
  3. Validator daemon generates a session wallet, simulates measurement collection, and constructs dual-key signed SessionProofs (logged, not submitted -- Economic contracts deferred)
  4. All three daemons share types via a monorepo shared package and use @mysten/sui SDK for chain interactions
