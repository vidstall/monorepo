<task type="auto">
  <name>Task 1: Pre-flight checks and testnet deployment</name>
  <files>.env.testnet, .planning/PROJECT.md</files>
  <action>
    Execute the full deployment sequence. Each step must succeed before proceeding to the next.

    **Step 1: Pre-flight checks**
    ```bash
    # Verify all tests still pass (gate)
    sui move test --silence-warnings

    # Verify active environment is testnet
    sui client active-env

    # If not testnet, switch:
    # sui client switch --env testnet

    # Verify active address matches expected
    sui client active-address
    # Expected: 0x3357c00f615f4cc1d89e0b580603ff9f474ee88bff89aecf587caf9075f5d306

    # Check gas balance
    sui client gas
    ```

    **Step 2: Request faucet gas (if balance is insufficient)**
    ```bash
    curl --location --request POST 'https://faucet.testnet.sui.io/v1/gas' \
      --header 'Content-Type: application/json' \
      --data-raw '{"FixedAmountRequest":{"recipient":"0x3357c00f615f4cc1d89e0b580603ff9f474ee88bff89aecf587caf9075f5d306"}}'
    ```
    If rate-limited (HTTP 429), wait 60 seconds and retry. Each request gives enough SUI for multiple publishes.

    **Step 3: Publish to testnet**
    ```bash
    sui client publish --gas-budget 100000000 --json
    ```
    Use `--json` for machine-parseable output per research recommendation.

    **Step 4: Extract object IDs from publish output**
    From the JSON output, extract:
    - `PackageID` from the Published Objects section
    - `NetworkRegistry` (shared object of type `::network_registry::NetworkRegistry`)
    - `MinerStore` (shared object of type `::miner_store::MinerStore`)
    - `TreasuryCap` (owned object of type `::token::...TreasuryCap`)
    - `AdminCap` (owned object of type `::network_registry::AdminCap`)
    - `UpgradeCap` (owned object of type `::package::UpgradeCap`)
    - Transaction digest from the response

    **Step 5: Create `.env.testnet`**
    Write the file at project root with format:
    ```
    # DVConf Testnet Deployment
    # Published: <date>
    # Transaction Digest: <digest>

    SUI_NETWORK=testnet
    PACKAGE_ID=0x...
    NETWORK_REGISTRY_ID=0x...
    MINER_STORE_ID=0x...
    TREASURY_CAP_ID=0x...
    ADMIN_CAP_ID=0x...
    UPGRADE_CAP_ID=0x...
    ```

    **Step 6: Update `.planning/PROJECT.md`**
    Add a "Testnet Deployment" section to PROJECT.md with the deployed object IDs and transaction digest.

    **If `sui client publish` fails:**
    - Gas error: Request more from faucet, retry
    - Compilation error: This should not happen if Plan 01 passed all tests. Investigate and fix.
    - Network error: Retry with same command
    - Wrong environment: `sui client switch --env testnet` and retry
