       - `package.json`: name `@dvconf/signaling`, `"type": "module"`, dependencies: `ws` ^8, `@dvconf/shared` (workspace:*) тАФ ONLY for logger, NOT for chain helpers. devDeps: `@types/ws` ^8. Scripts: `start`, `dev` (tsx watch).
       - `tsconfig.json`: extends `../../tsconfig.base.json`

    2. **Room manager (`src/rooms.ts`):**
       - `RoomManager` class with a `Map<string, Set<WebSocket>>` for room membership
       - `join(roomId, ws, peerId)`: adds ws to room set, broadcasts `peer-joined` to other peers in room
       - `leave(ws)`: removes ws from its room, broadcasts `peer-left`, cleans up empty rooms
       - `broadcast(roomId, sender, message)`: sends message to all peers in room except sender
       - `getRoomSize(roomId)`: returns number of peers
       - `getStats()`: returns { rooms: number, connections: number }

    3. **Signaling server (`src/index.ts`):**
       - Creates WebSocketServer on configurable port (env SIGNALING_PORT or 8080)
       - On connection: assigns a unique peerId (crypto.randomUUID())
       - Message protocol (JSON):
         - `{ type: "join", roomId: string }` тАФ join a room
         - `{ type: "offer", sdp: object, targetPeerId: string }` тАФ forward SDP offer to specific peer
         - `{ type: "answer", sdp: object, targetPeerId: string }` тАФ forward SDP answer
         - `{ type: "ice-candidate", candidate: object, targetPeerId: string }` тАФ forward ICE candidate
         - `{ type: "leave" }` тАФ leave current room
       - On close: clean up room membership
       - Graceful shutdown on SIGTERM/SIGINT (close WebSocketServer)
       - Use pino logger from @dvconf/shared тАФ but NO imports from shared/chain/* or @mysten/sui
       - Do NOT log ICE candidate contents (may contain private IPs per OFFCHAIN_AGENT_SKILL.md)

    4. **Tests:**
