<objective>
Build the Control Plane daemon that subscribes to Sui registry events, runs the relay scoring algorithm, sends heartbeat transactions to ControlPlaneRegistry, and uses exponential backoff on failures. Includes auto-registration on startup.

Purpose: The CP daemon is the on-chain-aware orchestrator that monitors relay nodes and computes scores for future relay assignment. In this phase it logs scores (voting deferred to v2 when Room lifecycle contracts exist) and demonstrates heartbeat liveness.

Output: Working CP daemon at `dvconf-daemons/apps/cp-daemon/` with passing unit tests.

    2. **Relay scoring (`src/scoring.ts`):**
       - `RelayCandidate` interface: minerId, reputation (bigint, 0-10000 basis points), rtt (bigint, ms), load (bigint, current connections), stakeAmount (bigint, MIST), region (string).
       - `ScoringWeights` interface: reputation, rtt, load, stake, regionMatch (all bigint, sum to 10_000).
       - `scoreRelay(relay: RelayCandidate, weights: ScoringWeights, targetRegion: string): bigint` — implements the scoring formula from spec rev4 Section 9.1:
         - repScore = relay.reputation (already 0-10000)
         - rttScore = inverse of RTT, capped at 10000 (lower RTT = higher score)
         - loadScore = inverse of load, capped (lower load = higher score)
         - stakeScore = normalized to 0-10000 range (e.g., linear scale with 10 DVCONF as max)
         - regionBonus = 10000 if same region, 0 otherwise
         - Final = (w_rep * repScore + w_rtt * rttScore + w_load * loadScore + w_stake * stakeScore + w_region * regionBonus) / 10000
       - All math in bigint — NO floating point. Division truncates (acceptable for scoring).
       - `scoreRelays(relays: RelayCandidate[], weights: ScoringWeights, targetRegion: string): Array<{minerId: string, score: bigint}>` — scores all relays, returns sorted descending by score.

    3. **Event handler (`src/event-handler.ts`):**
       - Maintains in-memory relay state: `Map<string, RelayCandidate>` — populated from events.
       - `handleEvent(event: SuiEvent, relayState: Map, logger)`:
         - `RelayRegistered`: add relay to state map with initial values
         - `RelayLoadUpdated`: update load for the relay
         - `RelayRTTUpdated`: update RTT for the relay
         - `RoomCreated`: log the room creation, run scoring against all known relays for that room's region, log the ranked scores. Do NOT submit votes.
         - Unknown event types: log and skip
       - Export `createEventHandler(config, logger)` that returns a handler function bound to its relay state map.

    4. **Tests:**

         - (a) Score with equal weights produces expected ranking

         - (c) Lower RTT produces higher score
         - (d) All-zero relay scores zero
         - (e) Weights must be bigint, results are bigint (no floating point)
         - (f) scoreRelays returns sorted descending
       - `__tests__/event-handler.test.ts`:
         - (a) RelayRegistered adds relay to state
         - (b) RelayLoadUpdated updates existing relay
         - (c) RelayRTTUpdated updates existing relay
         - (d) RoomCreated triggers scoring and logs results
         - (e) Unknown event type is logged and skipped
         - (f) Events for unknown relays are handled gracefully

Returns a stop function (clearInterval). Uses the shared TX wrapper which handles exponential backoff internally (DAEMON-07).
