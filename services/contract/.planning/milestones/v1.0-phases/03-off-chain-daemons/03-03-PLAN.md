    2. **Measurements (`src/measurements.ts`):**
       - `MeasurementResult` interface:
         - `relayMinerId: string` — which relay was measured
         - `packetsSent: bigint` — total packets sent through relay
         - `packetsReceived: bigint` — packets received (integrity check)
         - `packetLossRate: bigint` — loss in basis points (0-10000)
         - `avgLatencyMs: bigint` — average round-trip latency in ms
         - `jitterMs: bigint` — latency variation
         - `bytesForwarded: bigint` — total bytes the relay forwarded
         - `measurementDurationMs: bigint` — how long the measurement window was
         - `timestamp: bigint` — epoch timestamp
       - `collectMeasurements(relayMinerId: string): MeasurementResult` — generates simulated measurements with realistic ranges:
         - packetsSent: 10000-50000
         - packetsReceived: 95-100% of sent (simulating 0-5% loss)
         - avgLatencyMs: 20-150ms
         - jitterMs: 2-20ms
         - bytesForwarded: 1MB-100MB range
         - measurementDurationMs: 60000 (1 minute window)
       - All values are bigint. Use `BigInt(Math.floor(Math.random() * range))` for simulation.
       - Log: "Simulated measurement for relay {relayMinerId}: loss={lossRate}bp, latency={avgLatency}ms, bytes={bytesForwarded}"

    3. **SessionProof (`src/session-proof.ts`):**
       - `SessionProof` interface:
         - `roomId: string`
         - `relayMinerId: string`
         - `validatorMinerId: string`
         - `sessionWalletAddress: string`
         - `measurement: MeasurementResult`
         - `epoch: bigint`
       - `buildSessionProof(roomId, relayMinerId, validatorMinerId, sessionWalletAddress, measurement, epoch): SessionProof` — constructs the proof object.
       - `serializeProof(proof: SessionProof): Uint8Array` — deterministic serialization of proof fields to bytes (use JSON.stringify + TextEncoder for simplicity in this phase; production would use BCS).
       - `dualKeySign(proofBytes: Uint8Array, mainKeypair: Ed25519Keypair, sessionKeypair: Ed25519Keypair): { signatureA: Uint8Array, signatureB: Uint8Array }` — signs the proof bytes with both keypairs. Main wallet signs first (public validator identity), then session wallet signs (ephemeral identity). Uses `keypair.sign(proofBytes)` from @mysten/sui.
       - Log: "SessionProof constructed — room={roomId}, relay={relayMinerId}, loss={lossRate}bp. Dual-key signed. NOT submitting (Economic layer not yet deployed)."
       - CRITICAL: Never log private keys or session wallet secret keys.

  <action>
    1. **Auto-registration (`src/auto-register.ts`):**
       - `ensureRegistered(client, signer, config, logger): Promise<{ validatorCapId: string }>` — implements the Validator auto-registration flow:
         - Check env for `VALIDATOR_CAP_ID`. If set, return it immediately (already registered).
         - If NOT set, attempt auto-registration:
           a. Build TX calling `registration::register()` with role=Validator (MinerRole.Validator=1), passing required parameters: registry, store, coin (stake), endpoint fields. Use placeholder endpoint values for validators (validators don't serve media directly).
           b. Execute via `executeWithRetry`. If fails (insufficient funds, already registered), log error with clear message ("Auto-registration failed: ensure wallet has DVCONF tokens and SUI gas") and exit with process.exit(1).
           c. After successful miner registration, extract the created object IDs from TX effects.
           d. Build TX calling `validator_registry::register_validator()` with the miner cap.
           e. Execute via `executeWithRetry`. Extract VALIDATOR_CAP_ID from created objects.
           f. Log: "Auto-registered as Validator — cap ID: {validatorCapId}. Set VALIDATOR_CAP_ID={validatorCapId} in .env to skip registration on next startup."
           g. Return the validatorCapId.
       - CRITICAL: All TX calls go through `executeWithRetry` from @dvconf/shared (DAEMON-07 + DAEMON-12 compliance).

    2. **Entry point (`src/index.ts`):**
       - Loads env (dotenv), creates SuiClient, loads main keypair, loads network config
       - Calls `ensureRegistered(client, signer, config, logger)` to get validatorCapId — auto-registers if VALIDATOR_CAP_ID not in env, or uses the env value if present.
       - Generates session wallet: `generateSessionKeypair()` from @dvconf/shared — creates a fresh Ed25519Keypair. The session wallet address is what gets assigned on-chain via ValidatorRegistry (DAEMON-08).
       - Logs: "Validator daemon started — main wallet: {mainAddress}, session wallet: {sessionAddress}"
       - Starts a periodic measurement loop (configurable interval, default 60000ms):
         - Calls `collectMeasurements(relayMinerId)` with a placeholder relay ID (from env or "demo-relay")
         - Calls `buildSessionProof()` with the measurement
         - Calls `dualKeySign()` with main + session keypairs
         - Logs the proof summary. Does NOT submit on-chain (DAEMON-10 scoped).
       - Starts EventPoller for validator_registry module to listen for SessionWalletAssigned, SessionWalletRevealed events
       - Handles SIGTERM/SIGINT: stops measurement loop, stops event pollers, logs shutdown
       - CRITICAL: Never log session wallet private key. Only log the address.

    3. **Tests:**
       - `__tests__/auto-register.test.ts`:

         - (b) Calls registration::register then validator_registry::register_validator when VALIDATOR_CAP_ID not set
